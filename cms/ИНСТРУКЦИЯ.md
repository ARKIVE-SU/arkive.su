# 📋 ИНСТРУКЦИЯ: Как добавить статью на сайт ARKIVE.SU

Эта инструкция для контент-менеджеров. Вам нужен только браузер и подготовленный текст.

---

## 📦 Первый запуск (если ещё не делали)

1. **Установите Node.js** — скачайте с [nodejs.org](https://nodejs.org/) версию LTS, установите как обычную программу.
2. Откройте **Терминал** (Win + R → `cmd` → Enter).
3. Перейдите в папку проекта:
   ```
   cd путь\к\папке\ARKIVE.SU
   ```
4. Установите зависимости (один раз):
   ```
   npm install
   ```

---

## 🚀 Запуск CMS-панели

Каждый раз, когда нужно добавить статью:

1. Откройте **Терминал** и перейдите в папку проекта.
2. Запустите CMS:
   ```
   npm run cms
   ```
3. Откройте браузер и перейдите по адресу:  
   **http://localhost:3000**

Вы увидите панель управления статьями.

---

## 📝 Как создать статью

### Шаг 1 — Заполните общие поля

| Поле | Что писать | Пример |
|------|-----------|--------|
| **Slug** | URL-имя статьи, латиница через дефис. Заполняется автоматически из англ. заголовка | `ceramic-storage-future` |
| **Дата** | Дата публикации | `2026-03-11` |
| **Изображение** | Нажмите «Загрузить» и выберите картинку (макс. 10 МБ) | — |

### Шаг 2 — Заполните текст на каждом языке

Сверху находятся вкладки: 🇬🇧 English, 🇷🇺 Русский, 🇪🇸 Español, 🇨🇳 中文, 🇸🇦 العربية.

Для каждого языка заполните:

- **Заголовок** — название статьи
- **Краткое описание** — 1-2 предложения для ленты новостей
- **Контент** — полный текст статьи в формате Markdown

> 💡 **Подсказка:** Зелёная точка на вкладке означает, что этот язык заполнен.  
> Можно заполнить не все языки — файлы создадутся только для заполненных.

### Шаг 3 — Нажмите «🟢 Создать статью»

Файлы `.md` будут созданы в папках проекта. В консоли внизу появится отчёт.

---

## 🔨 Как опубликовать на сайте

После создания статьи нужно выполнить два действия:

### 1. Нажмите «🔨 Собрать сайт»

Сайт пересоберётся. В консоли будет виден процесс. Дождитесь сообщения `Build finished with code 0`.

### 2. Нажмите «🚀 Запушить на GitHub»

- Введите **название коммита** (описание изменения), например:  
  `feat: статья про керамическое хранение`
- Нажмите «Запушить»

Фсё отправится на GitHub. Дождитесь сообщения `Push finished with code 0`.

### 3. Обновите сервер

Подключитесь к серверу по SSH и выполните:

```bash
cd /var/www/www-root/data/www/arkive.su
git pull
```

Сайт обновится мгновенно!

## 📋 Управление статьями

Нажмите кнопку **«📋 Статьи»** в правом верхнем углу, чтобы увидеть список всех существующих статей. Оттуда можно удалить ненужную статью.

## ❓ Проблемы и решения

**Панель не открывается**  
→ Убедитесь, что в терминале запущена команда `npm run cms` и нет ошибок.

**Сборка завершилась с ошибкой**  
→ Проверьте файлы Markdown на ошибки в формате. Часто проблема в неправильных кавычках в заголовке.

**Git push не работает**  
→ Убедитесь, что Git настроен: `git config user.name "Ваше Имя"` и `git config user.email "ваш@email.com"`.

**На сервере `git pull` выдаёт ошибку «dubious ownership»**  
→ Выполните одну из команд:
```bash
git config --global --add safe.directory /var/www/www-root/data/www/arkive.su
```

**Сайт на сервере показывает 404 или Forbidden**  
→ Проверьте, что Document Root в ISPmanager указывает на `_site`:  
`/var/www/www-root/data/www/arkive.su/_site`
